# Domain-Driven Design (DDD) - Curriculum

## Module 1: Strategic DDD
- [ ] Ubiquitous Language - shared vocabulary between devs and domain experts
- [ ] Bounded Context - explicit boundaries for models
- [ ] Context Mapping: Shared Kernel, Customer-Supplier, Conformist, Anti-Corruption Layer
- [ ] Subdomains: Core, Supporting, Generic

## Module 2: Tactical DDD
- [ ] Entities: identity-based objects
- [ ] Value Objects: immutable, equality by value
- [ ] Aggregates: consistency boundary, aggregate root
- [ ] Repositories: persistence abstraction for aggregates
- [ ] Domain Events: something that happened in the domain
- [ ] Domain Services: logic that doesn't belong to an entity

## Module 3: DDD in Microservices
- [ ] One microservice per bounded context
- [ ] Event-driven communication between contexts
- [ ] Anti-corruption layer at service boundaries
- [ ] DDD with CQRS and Event Sourcing
- [ ] Implementing DDD in Spring Boot / Quarkus
