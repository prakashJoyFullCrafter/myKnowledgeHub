# Behavioural Design Patterns - Curriculum

## Topics
- [ ] **Strategy**: interchangeable algorithms at runtime
- [ ] **Observer**: notify dependents of state changes (pub/sub)
- [ ] **Template Method**: skeleton algorithm with overridable steps
- [ ] **Command**: encapsulate request as object
- [ ] **Chain of Responsibility**: pass request along handler chain
- [ ] **State**: behavior changes based on internal state
- [ ] **Iterator**: traverse collection without exposing internals
- [ ] **Mediator**: reduce coupling between objects
- [ ] **Visitor**: add operations to objects without modifying them
- [ ] Spring Boot: Strategy (multiple `@Service`), Observer (`@EventListener`), Template (`JdbcTemplate`)
