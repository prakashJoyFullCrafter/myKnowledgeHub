# CAP Theorem - Curriculum

## Topics
- [ ] C (Consistency): every read gets the most recent write
- [ ] A (Availability): every request gets a response
- [ ] P (Partition Tolerance): system works despite network failures
- [ ] You can only guarantee 2 of 3 (CP or AP, never CA in distributed systems)
- [ ] CP systems: MongoDB, HBase, Redis (with Sentinel)
- [ ] AP systems: Cassandra, DynamoDB, CouchDB
- [ ] PACELC theorem: extends CAP (what happens when no partition?)
- [ ] Eventual consistency vs strong consistency
- [ ] Real-world trade-off decisions
