# Event Storming - Curriculum

Alberto Brandolini's collaborative modeling workshop technique - rapidly explore complex domains with sticky notes and a wall.

## Topics
- [ ] What Event Storming solves - getting domain experts and devs aligned fast
- [ ] Materials: long wall, orange stickies, blue stickies, etc.
- [ ] Color conventions:
  - [ ] **Orange** - Domain Event (past tense)
  - [ ] **Blue** - Command
  - [ ] **Yellow** - Actor / User
  - [ ] **Lilac/Purple** - Policy ("whenever X happens, do Y")
  - [ ] **Pink** - External system
  - [ ] **Green** - Read model
  - [ ] **Red** - Hotspot / question / problem
- [ ] **Big Picture Event Storming** - explore the entire domain end-to-end
- [ ] **Process Modeling Event Storming** - zoom into one process
- [ ] **Software Design Event Storming** - design aggregate boundaries
- [ ] Workshop facilitation: who to invite, room layout, timing
- [ ] Identifying pivotal events ("swimlanes" / phases)
- [ ] Discovering Bounded Contexts from event clusters
- [ ] Discovering Aggregates from command + event + invariant patterns
- [ ] Common pitfalls: jumping to solutions, unrealistic timelines, too few domain experts
- [ ] Remote Event Storming with Miro / Mural / Lucidspark
- [ ] Output: Context Map + initial aggregate sketches + glossary
- [ ] Follow-up techniques: Domain Storytelling, Example Mapping, User Story Mapping
