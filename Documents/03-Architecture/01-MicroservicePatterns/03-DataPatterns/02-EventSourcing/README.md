# Event Sourcing - Curriculum

## Topics
- [ ] Store events, not current state
- [ ] Event store: append-only log of domain events
- [ ] Rebuilding state by replaying events
- [ ] Snapshots for performance (avoid replaying all events)
- [ ] Event versioning and schema evolution
- [ ] Event sourcing + CQRS combination
- [ ] Projections: building read models from events
- [ ] Benefits: complete audit trail, temporal queries, debugging
- [ ] Challenges: eventual consistency, complexity, storage growth
- [ ] Tools: Axon Framework, EventStoreDB
