# Data Patterns

Patterns for data management in distributed systems.

1. **CQRS** - Separate read/write models, variations, eventual consistency (4 modules)
2. **Event Sourcing** - Event store, projections, snapshots, schema evolution (5 modules)
3. **Outbox Pattern** - Dual-write problem, polling vs CDC, Debezium, idempotent consumers (5 modules)
4. **Strangler Fig** - Incremental migration, facade routing, anti-corruption layer, data decomposition (4 modules)
5. **Event-Driven Patterns** - Fowler's taxonomy: Event Notification, ECST, ES, CQRS — choosing between styles (6 modules)
