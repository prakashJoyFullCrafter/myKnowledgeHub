# Kafka vs RabbitMQ: Use Case Selection - Curriculum

## Topics
- [ ] Kafka: event streaming, log aggregation, event sourcing, high throughput
- [ ] RabbitMQ: task queues, RPC, routing flexibility, low latency
- [ ] Kafka: consumer pulls, persistent log, replay capability
- [ ] RabbitMQ: broker pushes, message acknowledged and removed
- [ ] Kafka: ordering per partition, at-least-once/exactly-once
- [ ] RabbitMQ: flexible routing, priority queues, per-message TTL
- [ ] Decision guide: choose Kafka when / choose RabbitMQ when
- [ ] Hybrid architectures: using both together
