# Code Review - Curriculum

## Topics
- [ ] Code review goals: correctness, readability, maintainability, knowledge sharing
- [ ] What to review: logic, edge cases, security, performance, naming, tests
- [ ] PR size: small and focused (< 400 lines)
- [ ] Review checklist: does it work, is it tested, is it readable
- [ ] Giving feedback: be kind, explain why, suggest alternatives
- [ ] Receiving feedback: don't take it personally, ask questions
- [ ] Automated checks: linting, formatting, security scanning
- [ ] Review tools: GitHub PR reviews, Gerrit
- [ ] Code review anti-patterns: rubber stamping, nitpicking, gatekeeping
