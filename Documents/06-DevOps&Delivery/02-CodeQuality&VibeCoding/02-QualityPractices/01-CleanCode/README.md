# Clean Code - Curriculum

## Topics
- [ ] Meaningful names: intention-revealing, pronounceable, searchable
- [ ] Functions: small, do one thing, one level of abstraction
- [ ] Comments: explain why, not what; avoid noise comments
- [ ] Formatting: consistent, team conventions
- [ ] Error handling: use exceptions, don't return null
- [ ] DRY, KISS, YAGNI principles applied
- [ ] Boy Scout Rule: leave code cleaner than you found it
- [ ] Code smells: long method, large class, feature envy, god class
- [ ] Refactoring techniques: extract method, rename, move
- [ ] Clean Code in Java/Kotlin specific patterns
