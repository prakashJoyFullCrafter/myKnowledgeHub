# Infrastructure as Code (Terraform) - Curriculum

## Topics
- [ ] IaC benefits: version-controlled, repeatable, reviewable infrastructure
- [ ] HCL syntax: `resource`, `variable`, `output`, `data`, `locals`
- [ ] Providers: AWS, GCP, Azure, Kubernetes
- [ ] State management: local, remote (S3, GCS), state locking
- [ ] `terraform init`, `plan`, `apply`, `destroy`
- [ ] Modules: reusable infrastructure components
- [ ] Workspaces: managing multiple environments
- [ ] `terraform import` for existing resources
- [ ] Terraform Cloud and Terraform Enterprise
- [ ] Best practices: small blast radius, state isolation, code review
- [ ] Alternatives: Pulumi, CloudFormation, Crossplane
