# Internal Developer Platform (IDP) - Curriculum

Self-service interface for app developers - abstracting cloud-native complexity.

## Topics
- [ ] What an IDP is and isn't (not just a wiki, not just a CI tool)
- [ ] Capabilities a mature IDP exposes:
  - [ ] Self-service environment provisioning
  - [ ] App scaffolding and onboarding
  - [ ] CI/CD pipeline templates
  - [ ] Deployment dashboards
  - [ ] Observability links
  - [ ] Secret management
  - [ ] Database / queue / cache provisioning
  - [ ] On-call / incident metadata
- [ ] **Backstage** (Spotify) - the dominant open-source IDP framework
  - [ ] Software Catalog (services, components, APIs, resources)
  - [ ] TechDocs - docs-as-code
  - [ ] Software Templates / Scaffolder
  - [ ] Plugins ecosystem
- [ ] Alternatives: Port, Cortex, Humanitec, Mia-Platform
- [ ] **Score** - workload specification standard
- [ ] **Crossplane** - IDP-style infra provisioning via K8s CRDs
- [ ] **Kratix** - platform-as-a-product framework
- [ ] Treating the platform as a product (PM, roadmap, customers)
- [ ] Platform team topology - thinly opinionated, thick on golden paths
- [ ] Measuring IDP success: DORA metrics, time-to-first-deploy, NPS
- [ ] Common failure modes: building too much, ignoring DevEx, ivory-tower platform
