# E2E Testing (Playwright, Cypress) - Curriculum

## Topics
- [ ] What E2E covers - real browser, real user flows
- [ ] **Playwright**: cross-browser (Chromium, WebKit, Firefox), parallel by default
- [ ] **Cypress**: developer experience, time-travel debugging
- [ ] Playwright vs Cypress - choosing in 2025
- [ ] Page Object Model (POM) pattern
- [ ] Selectors: data-testid, role, text - stability ranking
- [ ] Auto-waiting and retry-ability
- [ ] Network interception and stubbing
- [ ] Authentication: storage state, programmatic login
- [ ] Visual regression testing (Percy, Chromatic, Playwright snapshots)
- [ ] Running on CI: GitHub Actions, sharding, retries on failure
- [ ] Headed vs headless mode
- [ ] Trace viewer (Playwright) for debugging
- [ ] Test pyramid: many unit, fewer integration, fewest E2E
- [ ] Flakiness: causes and mitigations
