# Component Testing (React Testing Library) - Curriculum

## Topics
- [ ] Philosophy: test behavior, not implementation
- [ ] `render()`, `screen`, `cleanup`
- [ ] Queries: `getBy`, `queryBy`, `findBy` and their `*All*` variants
- [ ] Query priority: role > label > placeholder > text > testid
- [ ] `userEvent` vs `fireEvent` (prefer `userEvent`)
- [ ] Testing form inputs, clicks, keyboard interactions
- [ ] Async assertions: `findBy*`, `waitFor`
- [ ] Testing components with Context providers (custom `render` wrapper)
- [ ] Testing components with React Router
- [ ] Testing components with React Query
- [ ] Mocking child components vs integration testing
- [ ] Testing custom hooks with `renderHook`
- [ ] Accessibility queries enforce a11y by default
- [ ] Anti-patterns: testing internal state, snapshot-only tests
