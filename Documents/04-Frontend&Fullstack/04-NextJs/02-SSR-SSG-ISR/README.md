# SSR / SSG / ISR - Curriculum

## Topics
- [ ] **SSR** (Server-Side Rendering): render on every request
- [ ] **SSG** (Static Site Generation): render at build time
- [ ] **ISR** (Incremental Static Regeneration): regenerate static pages on demand
- [ ] App Router: Server Components are SSR by default
- [ ] `generateStaticParams()` for SSG with dynamic routes
- [ ] `revalidate` option for ISR (`export const revalidate = 60`)
- [ ] On-demand revalidation: `revalidatePath()`, `revalidateTag()`
- [ ] `fetch()` caching: `cache: 'force-cache'`, `cache: 'no-store'`
- [ ] Streaming SSR with Suspense
- [ ] When to use SSR vs SSG vs ISR vs CSR
